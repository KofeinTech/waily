# WAIL-14 Env Config Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a typed, immutable env config object accessible everywhere via `kEnv` / `kEnvHelper`, loaded from a single `.env` at the project root, with a dev-only banner overlay.

**Architecture:** `lib/core/env/` library with three `part of`-linked files (parser extension, typed variables, sealed union). `flutter_dotenv` loads `.env` from the asset bundle in `main()` before `runApp`. A freezed sealed union `_Env` with `_Dev` / `_Prod` variants is selected by the `TYPE` value inside `.env`. A `DevEnvBanner` wrapper in `MaterialApp.builder` renders Flutter's built-in `Banner` only when `kEnvHelper.isDev`.

**Tech Stack:** Flutter (FVM), `flutter_dotenv`, `equatable`, `freezed` (already in project), `flutter_test`, build_runner.

**Spec:** `docs/superpowers/specs/2026-04-27-wail-14-env-config-design.md`

Replace the entire content of `lib/core/env/env.dart` with:

```dart
import 'package:equatable/equatable.dart';
import 'package:flutter/foundation.dart';
import 'package:flutter_dotenv/flutter_dotenv.dart';
import 'package:freezed_annotation/freezed_annotation.dart';

part 'env.freezed.dart';
part 'env_parser.dart';
part 'env_variables.dart';

Future<void> initEnv() async {
  if (_parser.isInitialized) return;
  await _parser.load();
}

_EnvData get kEnv => kEnvHelper.data;

_Env get kEnvHelper => _buildEnv();

_Env? _envInstance;

_Env _buildEnv() {
  assert(_parser.isInitialized, 'kEnv not initialized, call initEnv() first');
  return _envInstance ??= switch (_EnvVariables.envType.toUpperCase()) {
    'PROD' => _Env.prod(_env),
    _ => _Env.dev(_env),
  };
}

@freezed
sealed class _Env with _$Env {
  const factory _Env.dev(_EnvData data) = _Dev;
  const factory _Env.prod(_EnvData data) = _Prod;
}

extension EnvX on _Env {
  bool get isDev => this is _Dev;
  bool get isProd => this is _Prod;

  _EnvData get data => switch (this) {
    _Dev(:final data) => data,
    _Prod(:final data) => data,
  };
}

@visibleForTesting
void resetEnvForTesting() {
  _envInstance = null;
  _parser.clean();
}
```

Notes for the implementer:
- The sealed union uses freezed 3.x syntax (`sealed class _Env with _$Env`).
- Logic that constructs the union is in a top-level `_buildEnv()` function rather than a non-redirecting factory inside `_Env`, because freezed 3.x is stricter about mixing redirecting and non-redirecting factories â€” keeping the build logic outside is safer and equally correct.
- `data` is exposed via the `EnvX` extension's switch over the sealed variants, so `kEnv` reads `kEnvHelper.data` cleanly.

## Environment configuration

The app reads runtime configuration from a single `.env` file in the project root. The file is gitignored â€” each developer keeps their own copy.

`.env.example` is the committed template. To get started:

```bash
cp .env.example .env
```

### Keys

| Key              | Type   | Description                                                |
|------------------|--------|------------------------------------------------------------|
| `TYPE`           | string | `DEV` or `PROD`. `DEV` shows the corner banner; any other value is treated as dev. |
| `API_BASE_URL`   | string | Base URL of the Waily backend.                             |
| `ENABLE_LOGGING` | bool   | `true` enables verbose logging; `false` (or missing) disables it. |

### Reading values in code

```dart
import 'package:waily/core/env/env.dart';

final url = kEnv.apiBaseUrl;
final loggingOn = kEnv.enableLogging;
final isDev = kEnvHelper.isDev;
```

### Adding a new key

1. Add a typed getter to `lib/core/env/env_variables.dart` (`_EnvVariables`).
2. Add the field to `_EnvData` (constructor, field, `props`).
3. Wire it into the `_env` getter at the bottom of the same file.
4. Update `.env.example` with a placeholder.

### Hot reload caveat

`flutter_dotenv` reads `.env` from the asset bundle. After editing `.env`, you must perform a hot **restart** (capital R in `flutter run`) â€” hot reload (`r`) does not re-read the file.

## Testing
